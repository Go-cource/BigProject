# Это большой проект на ЯП Go для управления удаленными компьютерами

## Описание проекта

Модуль состоит из _клиентской_ и _серверной_ частей.

- Сервер хранит базу данных удаленных компьютеров, а также ведет учет посталвенных задач.
- Клиент обращается на сервер за новыми заданиями, выполняет их и возвращает ответ.

## Структура серверной части

**Структура проекта:**

- `cmd/` - точка входа приложения
  - `main.go` - главный файл
- `internal/` - внутренние пакеты
  - `db/` - работа с базой данных
  - `handlers/` - HTTP обработчики
  - `models/` - модели данных
- `web/` - веб-интерфейс
  - `templates/` - HTML шаблоны
- `go.mod` - зависимости Go
- `README.md` - документация

## Описание БД

### Структура базы данных

#### **comps** - Таблица устройств/компьютеров

- `comp_id` - Уникальный идентификатор устройства (Первичный ключ, автоинкремент)
- `comp_name` - Имя хоста устройства
- `comp_last_time_message` - Временная метка последнего сообщения

#### **tasks** - Очередь задач

- `tasks_id` - Уникальный идентификатор задачи (Первичный ключ, автоинкремент)
- `tasks_comp` - Имя целевого устройства
- `tasks_creation_time` - Временная метка создания задачи
- `tasks_text` - Описание задачи/команда
- `tasks_result` - Результат выполнения
- `tasks_result_time` - Временная метка получения результата

### Рабочий процесс:

1. Устройство регистрируется в таблице `comps`
2. Задачи создаются в таблице `tasks` с указанием `tasks_comp`
3. Устройство опрашивает свои задачи, выполняет их
4. Результаты сохраняются обратно в таблицу `tasks`
